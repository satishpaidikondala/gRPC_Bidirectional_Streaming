# API Testing Guide

This guide provides detailed instructions on how to test the **gRPC Chat Service** using the provided Postman collections.

## 📥 Getting Started

1. Ensure the application is running: `docker-compose up --build -d`
2. Open Postman.
3. In the top-left corner, click **File -> Import**.
4. Select both `rest_collection.json` and `grpc_collection.json` from the `docs/postman/` directory.

---

## 🌐 REST API Testing
*Files: `docs/postman/rest_collection.json`*

### 1. Create a Room
- Open the **Create Room** request.
- Click **Send**.
- **Crucial**: Copy the `roomId` (the UUID) from the response JSON.

### 2. List Rooms
- Open the **List Rooms** request.
- Click **Send**.
- You will see all rooms and the number of active users in each.

### 3. Get Room History
- Open the **Get Room History** request.
- Go to the **Params** tab and look for **Path Variables**.
- Paste your copied `roomId` into the **Value** column for `roomId`.
- Click **Send** to see the last 100 messages.

---

## ⚡ gRPC API Testing
*Files: `docs/postman/grpc_collection.json`*

### 1. Setup Service Definition
*Postman needs the `.proto` file to understand the gRPC structure.*
- Open any gRPC request (e.g., **ChatStream**).
- Click the **Service definition** tab.
- Click **Import a .proto file**.
- Select `src/main/proto/chat.proto` from the project directory.
- Go back to the top and ensure the method is selected (e.g., `chat.ChatService/ChatStream`).

### 2. Connection
- Ensure the URL is `localhost:9090` (No http://).
- Click **Invoke** (or **Connect**). 
- The status should change to **"STREAMING"**.

### 3. Sending/Receiving Messages
- Click the **Message** tab.
- Ensure the dropdown on the right is set to **JSON**.
- Paste your test data:
  ```json
  {
      "room_id": "PASTE_YOUR_ROOM_ID_HERE",
      "user_id": "user1",
      "username": "Alice",
      "content": "Hello everyone!",
      "type": "TEXT"
  }
  ```
- Click **Send**.
- **View Responses**: Scroll down to the **Response** section. Click the timestamps in the stream to see the data (including the server-side `message_id` and `timestamp`).

---

## 🛠️ Payload Reference

| Method | Field | Description |
| :--- | :--- | :--- |
| **All** | `room_id` | The UUID generated by the REST "Create Room" API. |
| **ChatStream** | `content` | The message text to send. |
| **ChatStream** | `type` | Must be `TEXT` or `IMAGE_URL` for persistence. |

---

## 💡 Troubleshooting
- **Error: Parse Error**: Remove `http://` from the gRPC URL.
- **Empty Fields**: Ensure you use `snake_case` (e.g., `room_id`) in your JSON keys, not camelCase.
- **Connection Refused**: Ensure the Docker container is running and healthy.
