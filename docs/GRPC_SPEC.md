# gRPC API Specification

This document provides the technical details for interacting with the **ChatService** gRPC API.

## 📡 Service Overview
- **Server Address**: `localhost:9090`
- **Protocol**: gRPC over HTTP/2
- **Proto File**: `src/main/proto/chat.proto`

---

## 🛠️ Service Methods

### 1. JoinRoom
*Establish a server-side stream to receive live messages from a specific room.*

- **Type**: Server Streaming
- **Request**: `JoinRoomRequest`
- **Response Stream**: `ChatMessage`
- **Behavior**: Upon joining, the server broadcasts a "SYSTEM_JOIN" message to all participants in the room.

**Sample Request Body:**
```json
{
    "room_id": "4e1d5877-3cf9-4884-867e-cfa9e4487257",
    "user_id": "user1",
    "username": "Alice"
}
```

---

### 2. ChatStream
*Bidirectional streaming for real-time, two-way message exchange.*

- **Type**: Bidirectional Streaming
- **Request Stream**: `ChatMessage`
- **Response Stream**: `ChatMessage`
- **Persistence**: Messages of type `TEXT` and `IMAGE_URL` are persisted to the PostgreSQL database.

**Sample Chat Message:**
```json
{
    "room_id": "4e1d5877-3cf9-4884-867e-cfa9e4487257",
    "user_id": "user1",
    "username": "Alice",
    "content": "Hello everyone!",
    "type": "TEXT"
}
```

---

### 3. GetRoomUsers
*Retrieve a list of all currently active users in a specific room.*

- **Type**: Unary
- **Request**: `GetRoomUsersRequest`
- **Response**: `RoomUsersResponse`

**Sample Request Body:**
```json
{
    "room_id": "4e1d5877-3cf9-4884-867e-cfa9e4487257"
}
```

---

### 4. LeaveRoom
*Gracefully signal the server that a user is leaving the room.*

- **Type**: Unary
- **Request**: `LeaveRoomRequest`
- **Response**: `Empty`
- **Behavior**: The server broadcasts a "SYSTEM_LEAVE" message to the room and removes the user from the active count.

**Sample Request Body:**
```json
{
    "room_id": "4e1d5877-3cf9-4884-867e-cfa9e4487257",
    "user_id": "user1"
}
```

---

## 📦 Data Models

### ChatMessage
| Field | Type | Description |
| :--- | :--- | :--- |
| `message_id` | `string` | Unique identifier (UUID generated by server). |
| `room_id` | `string` | Target Room ID. |
| `user_id` | `string` | Sender's unique ID. |
| `username` | `string` | Sender's display name. |
| `content` | `string` | Message body. |
| `timestamp` | `int64` | Milliseconds since epoch (Server-side). |
| `type` | `enum` | `TEXT`, `IMAGE_URL`, `SYSTEM_JOIN`, `SYSTEM_LEAVE`. |

### MessageType (Enum)
- `TEXT`: Standard chat message.
- `IMAGE_URL`: Message containing an image link.
- `SYSTEM_JOIN`: Notification of a user entering.
- `SYSTEM_LEAVE`: Notification of a user exiting.
